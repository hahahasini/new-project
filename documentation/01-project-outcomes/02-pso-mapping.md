## Overview

Program Specific Outcomes (PSOs) are discipline-specific competencies that graduates of a PLACEHOLDER-BRANCH program are expected to demonstrate. Unlike the generic PO framework, PSOs reflect the domain-specific technical depth and specialized skill sets that distinguish the program. This document maps PLACEHOLDER-PROJECT-NAME to 6 PSOs, demonstrating how the project develops competencies in software development, machine learning, data engineering, and systems integration.

| PLACEHOLDER-PROJECT-NAME | **PSO1: Software Development Competency** — Ability to design, develop, test, and deploy software systems using modern programming languages, frameworks, and development methodologies. | **H** | The project demonstrates full-stack software development competency across Python and JavaScript ecosystems. The backend is built with **FastAPI** (Python 3.9+), implementing an asynchronous web server with structured routing (`APIRouter`), Pydantic schema validation (`AnalysisResponse`, `HealthCheckResponse`, `PredictionScore`, `Meals`, `DietDay`, `DeficiencyDietPlan`), middleware configuration (CORS with configurable origin whitelist), and lifecycle management (model loading at startup via `asynccontextmanager`). The frontend is a **React 18** single-page application scaffolded with **Vite**, employing modern React patterns including functional components with hooks (`useState`, `useEffect`, `useRef`), the Context API for global theme state (`ThemeProvider`/`useTheme`), `react-router-dom` v7 for declarative client-side routing with nested layouts (`<Outlet />`), and Axios for HTTP communication. The codebase follows separation of concerns — services (`predictor.py`, `diet_planner.py`, `api.js`), models (`schemas.py`), routers (`analysis.py`), utilities (`image_processing.py`), components, pages, and contexts are organized into dedicated modules. |
| PLACEHOLDER-PROJECT-NAME | **PSO2: Machine Learning & AI Competency** — Ability to apply machine learning algorithms, neural network architectures, and data-driven techniques to solve real-world problems. | **H** | The project implements three custom Convolutional Neural Networks for medical image classification, each fine-tuned using transfer learning on pre-trained architectures (MobileNetV2/ResNet50). The ML pipeline covers the full lifecycle: **Data Preparation** — clinical image datasets segmented by body part and disease class, with augmentation (rotation, flipping, zooming) to address class imbalance; **Model Training** — conducted in Jupyter notebooks (`nail disease.ipynb`, `skin disease.ipynb`, `tongue.ipynb`) using TensorFlow/Keras with categorical cross-entropy loss and the Adam optimizer; **Inference Optimization** — models are serialized as `.keras` files, loaded once at server startup, and execute predictions on preprocessed 224×224 RGB images normalized to [0,1] float32 with batch dimension expansion; **Evaluation** — a dedicated `evaluate_accuracy.py` script computes per-class accuracy against held-out test sets (Nail: 134 samples across 3 classes, Tongue: 76 samples across 2 classes, Skin: 280 samples across 2 classes), achieving up to 98.57% accuracy (Skin—Acne class). The inference pipeline achieves 92–145ms average latency, significantly outperforming cloud-based LLM alternatives (5–15s). |
| PLACEHOLDER-PROJECT-NAME | **PSO3: Data Engineering & Management** — Ability to design data models, manage data pipelines, and implement efficient data storage and retrieval mechanisms. | **M** | The project demonstrates competency in structured data management through several mechanisms: (1) **Configuration-driven data architecture** — the `config.py` module defines a comprehensive domain data model mapping body parts to disease classes (`DISEASES`), vitamin deficiency labels (`CLASSES`), and a 14-category food recommendation matrix (`FOOD_RECOMMENDATIONS`) with structured meal types (breakfast/lunch/dinner), enabling deterministic data retrieval without a traditional database; (2) **Data pipeline design** — the image analysis pipeline implements a well-defined ETL flow: raw image bytes → NumPy array decoding (via `cv2.imdecode`) → color space transformation (BGR→RGB) → spatial resizing (224×224) → normalization (float32/255) → batch expansion → model inference → argmax classification → deficiency/disease mapping → diet plan generation; (3) **Pydantic schema enforcement** — all API request and response data is validated through Pydantic models, ensuring type safety and serialization consistency; (4) **Data transformation** — the `DietPlannerService` implements a cyclic meal assignment algorithm that maps deficiency strings to 7-day weekly plans using modular indexing (`foods["breakfast"][i % len(foods["breakfast"])]`). |
| PLACEHOLDER-PROJECT-NAME | **PSO4: Systems Integration & API Design** — Ability to integrate heterogeneous systems, design RESTful APIs, and implement inter-service communication protocols. | **H** | The project integrates five distinct technology layers through well-defined interfaces: (1) **React ↔ FastAPI** — the frontend communicates with the backend via RESTful HTTP endpoints, using `multipart/form-data` for image upload and JSON for response consumption; (2) **FastAPI ↔ TensorFlow** — the backend's `PredictorService` abstracts model loading and inference behind a class-level API, decoupling the routing layer from ML framework specifics; (3) **React ↔ OpenStreetMap** — the `NearbyDoctors` component integrates with the Overpass API (OpenStreetMap's query endpoint) to fetch hospital/clinic data within a configurable radius, processing the response into structured marker data for Leaflet map rendering; (4) **React ↔ Browser APIs** — the application integrates the Geolocation API (`navigator.geolocation.getCurrentPosition`) with configurable accuracy and timeout parameters, and the File API for drag-and-drop image handling; (5) **React ↔ Chart.js** — prediction scores are piped into Chart.js via `react-chartjs-2` for real-time bar chart rendering. The API design follows REST principles with proper HTTP status codes (200, 400, 401, 500), content-type validation, and structured error responses. |
| PLACEHOLDER-PROJECT-NAME | **PSO5: User Interface & Experience Design** — Ability to design and implement user-centered interfaces that are accessible, responsive, and aesthetically refined. | **H** | The frontend demonstrates advanced UI/UX competency: (1) **Responsive design** — the CSS implements a comprehensive design system with CSS custom properties (variables for colors, spacing, typography, border radii), media queries for mobile/tablet/desktop breakpoints, and a flexible layout system using CSS Grid (model cards, diet plan grid) and Flexbox (navbar, result items); (2) **Theme system** — a complete dark/light mode toggle implemented via CSS `[data-theme]` attribute selectors and React Context, providing consistent visual theming across all components; (3) **Accessibility** — ARIA labels on interactive elements (`aria-label`, `aria-selected`, `aria-expanded`, `aria-hidden`), semantic HTML (`<main>`, `<header>`, `<nav>`, `<article>`, `<section>`), keyboard navigation support (`onKeyDown` handlers on dropzones), and role attributes (`role="tablist"`, `role="tab"`, `role="tabpanel"`); (4) **Micro-interactions** — CSS animations (`animate-fade-in`, `animate-fade-in-up`, `animate-slide-in-right`, `stagger-children`), skeleton loading states during analysis, spinner indicators, glassmorphism effects (`.glass` class), and confidence-colored indicators; (5) **Component composition** — reusable UI components (`ImageUploader`, `ConfidenceChart`, `DietPlan`, `NearbyDoctors`, `ResultsPanel`, `Navbar`, `Footer`) following React composition patterns. |
